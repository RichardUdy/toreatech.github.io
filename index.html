Logbook Support

Quick Start Guide

Create a new Logbook

Tap the "+" button, and enter a name for the Logbook.

Add fields to the Logbook

  1. Tap the "i" button, then tap the "+" button to show the Field form.
  2. Choose the type of field.
  3. Enter the details required for the field, and tap "Save" (this button is enabled once all required details are supplied).

Rearrange the Logbook fields (optional)

  1. In the Fields list, tap the "Edit" button, then use the handles to the right of each field to drag them into the order you want them.
  2. When done, tap the "Tick" button.

Add log entries

  1. Tap the Logbook to open it, then tap the "+" button to show the log entry form.
  2. Enter/select values for the fields you want to record, and tap "Save".
  3. The Logbook shows its entries, with the latest at the top.

Tip:
To quickly add a new entry without opening the Logbook, simply long-press the Logbook until it turns yellow, then enter the details and save it.

Review a Logbook's entries

  1. Tap the Logbook to open it.
  2. The entries are listed, with the latest at the top, each showing a summary of the recorded details.
  3. Tap an entry to view it in detail. From here you can:
    • Swipe back and forwards through the entries.
    • Amend recorded values by tapping on them and saving the changes.


Field Types Guide

Text

Used for text up to 40 characters. For longer text, use the Notes section at the bottom of the record, which can wrap longer text over multiple lines.

Numeric

Used for recording numeric values.
You also need to specify the unit, i.e. what the number refers to. The unit would generally be a symbol or abbreviation (e.g. %, kg, $). When displaying the value, currency symbols are generally shown immediately before, and others immediately after the value. If you want a space between the number and the unit, add a space when specifying the unit (e.g. " kg").
When entering the value you can specify as many decimal places as you want. The Decimal Places setting for the field is used only when displaying the value, and does not affect your stored value. You can change the Decimal Places setting at any time to show more or less of the recorded value.

Picker List

Used to pick the field value from a pre-defined list.
You can create a new Picker List as part of specifying the field, or you can re-use a list that has already been defined.
Pre-fill Logic allows you to set the default selection when adding a new entry: either the first item in the list, or the same item that was used in the previous entry.

Numeric Picker (Number Wheel)

Used to record a numeric value by selecting it from a number wheel, rather than entering the digits. It requires the Unit to be specified (as with Numeric fields), and also has Pre-fill Logic (as with Picker List fields). This field type can be useful where the value in successive entries may be the same or close to that of the previous entry.
This field specifies a range of numbers (e.g. -10 through to 20), and a Step Size which is the difference between successive numbers. Normally the numbers are listed with the lowest at the bottom, and the highest at the top. To reverse this order, use a negative Step Size (e.g. -2).
Unlike a Picker List field, the default selection is 0, or as close to 0 as the list allows. You can override this by setting Pre-fill Logic to "Previous".
You can change the range and Step Size at any time without affecting existing entries - their value will be temporarily added to the number wheel if necessary.

Time

Used to record a period of time. This field type comes in two forms, which you choose when creating the field, and which cannot be changed later:

  • Event Duration - starts a timer, and provides a button to finish the timer. Tapping the button shows the period of time since the entry was created. This time value can be edited in the same way as other field types.
  • Elapsed Time - shows the time since the previous entry. As the period shown depends on the timestamps of the two entries, amending one of the timestamps will affect the value.


FAQ

How do I get rid of a Logbook?

Slide the Logbook to the left, and tap "Hide". You can view hidden Logbooks by tapping "Recover Logbooks" button at the bottom of the list. From there you can either restore (i.e. Unhide) a Logbook, or permanently delete it.

Can I change the name of a Logbook?

No, once you have named a Logbook, it cannot be changed.

Can I rearrange the list of Logbooks?

Yes. Tap "Edit", then use the grab handles on the right edge to drag items where you want them. Tap the Tick button when done.

Can I remove fields that I don't need?

Yes. Slide the field to the left, and tap "Hide". Use the Hidden Fields button to view any hidden fields - from there you can either restore them, or permanently delete them.

Can I change how a field is specified?

You cannot change the internal name, or the field type. But all the other attributes can be changed. On the fields list, tap on the field to make it editable, then tap "Save" when done.

Why is the "Save" button grey-ed out?

The "Save" button is generally disabled until all required values have been entered.

How do I define a Picker List?

A Picker List is generally defined when specifying the first field that uses it. When you specify a Picker List field, you choose from a list of available Picker Lists, which includes a "New List" item. Choosing "New List" adds a field for you to name the new Picker List. Once you have saved the field settings, tap the new Picker List name. This opens a view where you can specify the items in the list, and arrange them in an appropriate order.

How can I manage existing Picker Lists?

Tap the Admin Menu button (at the top left of the list of logbooks), then tap "Picker Lists". This shows the Picker Lists that are currently being used, with the Logbooks using them, and below that a list of unused Lists. Unused lists can be deleted (by sliding left).

Can I get rid of unwanted Logbook entries?

Yes, by sliding the entry to the left.

Can I change an entry's timestamp?

Yes. Tap the entry in the list to open it in its detail view. Then tap the timestamp to show it as a date and time picker. Tap "Save" to update the entry. You may need to go back and re-open the Logbook to ensure the entries are still in timestamp order.

Can I export a Logbook's data?

A Logbook's data can be exported as a .CSV file (comma-separated-variables), which can be opened by most spreadsheet applications. Open the Logbook, then tap the "Folders" button at the top. Tap "Export", then choose "Save to Files" from the various options offered, and choose where to save the file. Exporting the data does not affect the Logbook's entries in any way.

Where is the Logbook data held?

The data is stored only on your device where you use the app, and in your iCloud backup if you have that enabled. There is no remote storage.

Can I set the app back to its Factory Settings?

The Admin Menu (top left above the list of Logbooks) provides the "Factory Reset" option.
Use this with CAUTION.
If you choose this option, and confirm to Reset the Settings, all the settings and changes you have made in the app will be wiped, and the app returned to the state it was in when first installed.
This action cannot be undone!
